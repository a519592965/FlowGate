# FlowGate v1.0.3

这次版本把 Plus / PayPal 支付链路里的几个易错点都补齐了，重点是把“提醒”和“恢复动作”真正落到流程里。

## 本次更新

- 第 6 步的 PayPal 验证码错误，不再一错就整段重开。
- 验证码输入错误、接口暂时拿不到码时，会先在当前步骤内重试。
- 连续失败 3 次后，才重新开始这一段验证流程。
- PayPal 验证页新增 “Get a new code” 这类错误识别。
- 支付页地址不被识别时，会直接回收到地址重填/换地址流程，不再继续硬提交。
- 支付页税务/地址识别提醒现在会被记录进流程状态，便于后台继续做恢复。
- 验证码接口现在可以兼容 JSON、纯文本、HTML 报错页和嵌套字段。

## 说明

- 看到 “The customer's location isn't recognized” 时，先换一条可识别地址，再继续。
- 看到 “Sorry, something went wrong. Get a new code.” 时，先重新获取验证码或换号，再继续。
- 如果同一段里连续失败 3 次，系统才会重新开始这一段流程。
- 版本号已同步为 `1.0.3`。
