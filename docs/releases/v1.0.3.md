# FlowGate v1.0.3

本版本修正 Plus 支付链路里的代理作用域，让流程页面代理继续作为主代理，只在支付转换请求那一下临时切换。

## 本次更新

- 流程页面代理不再整体绕开 `plus-checkout-create`，创建支付页、打开 ChatGPT、读取 session 和后续 checkout 页面仍按流程页面代理执行。
- 本地支付转换代理只包住创建 checkout session 的请求，结束后立即恢复到流程页面代理。
- 云端支付转换会把支付转换代理地址随请求传给云端服务，由云端在转换 OpenAI checkout 请求时临时使用。
- 更新侧边栏说明，明确流程页面代理是全流程页面主代理，支付转换代理只负责转换请求。
- 将扩展版本更新到 `1.0.3`。

## 说明

- 如果只配置流程页面代理，不配置支付转换代理，支付转换会沿用当前流程页面代理或云端服务默认出口。
- 如果同时配置两者，页面访问始终回到流程页面代理，支付转换请求短暂使用支付转换代理。
